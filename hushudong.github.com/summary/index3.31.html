(二)下午所学总结

  • 1.git push -u origin master # 将本地主分支推到远程(如无远程主分支则创建，用于初始化远程仓库)。
  • 2.git add .:提交所有文件/文件夹到待提交文件.文件夹中。
  • 3.git commit -m “描述”:添加项目描述。
  • 4.git push origin master:提交到云端版本库。
  • 5.git pull --no-ff # 抓取远程仓库所有分支更新并合并到本地，不要快进合并。
  • 6.git remote -v # 查看远程服务器地址和仓库名称。
  • 7.git remote add origin git@ github:robbin/robbin_site.git # 添加远程仓库地址.
  • 8.git push -u origin develop # 首次将本地develop分支提交到远程develop分支，并且track。
  • 9.git remote set-head origin master # 设置远程仓库的HEAD指向master分支。
  • 10.git push origin local_branch # 创建远程分支， origin是远程仓库名.
  • 11.git remote set-head origin master # 设置远程仓库的HEAD指向master分支。
  • 12.git merge origin/master # 将远程主分支合并到本地当前分支。
